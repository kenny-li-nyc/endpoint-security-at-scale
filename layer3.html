Layer 03

Detection & Analytics

Turning raw telemetry into alerts that analysts can act on.

Foundation

What detection engineering does

Telemetry is raw data. Detection is the process of identifying which patterns in that data represent a real threat.

Detection engineers write rules that match known attack patterns. They tune those rules to reduce false positives. They map detections to the MITRE ATT&CK framework so analysts understand what technique is being used.

At enterprise scale, manual triage is not possible. Alerts must be prioritized automatically. High fidelity detections go directly to analysts. Low fidelity signals get correlated before surfacing.

Core platforms

Detection tools in the enterprise

Microsoft Defender for Endpoint

Defender for Endpoint is an EDR platform for Windows. It provides behavioral detection and process tree visibility.

Microsoft Sentinel

Sentinel aggregates logs from endpoints, identity systems, and cloud infrastructure into a central workspace. It runs KQL based detection rules and automatically correlates related alerts into incidents for analyst triage.

KQL (Kusto Query Language)

KQL is the query language for Defender and Sentinel. Analysts use it to hunt threats and build detection rules.

MITRE ATT&CK Framework

Microsoft and the security research community document adversary tactics and techniques from incidents. Teams use this to classify detections.

Microsoft Defender for Identity

Monitors Active Directory for identity based attacks. Detects pass the hash, Kerberoasting, and lateral movement. Integrates with Defender for Endpoint for unified incident views.

Microsoft Defender XDR

Defender XDR is the unified portal correlating signals across Defender for Endpoint, Identity, and other Microsoft security products. It aggregates related alerts into a single incident view so analysts work one case instead of many disconnected alerts.

Microsoft Defender Threat Intelligence

MDTI provides adversary profiles, attack infrastructure data, and indicator context that enrich detections in Sentinel and Defender XDR. It connects alerts to known threat actor campaigns and helps analysts understand the scope of an attack faster.

Detection logic

How detection rules work

A detection rule is a query that runs continuously against telemetry. When a match occurs, it creates an alert.

Good detection rules are specific enough to avoid false positives. They are broad enough to catch real variations of an attack. Writing them requires understanding both the attacker technique and normal system behavior.

Behavioral detection goes further than signature matching. Instead of looking for a known malicious file, it looks for suspicious sequences. PowerShell spawning a network connection is unusual. A service stopping and immediately restarting is unusual. These patterns trigger investigation even without a known malware signature.

Pipeline connection

How this layer feeds the next

Detection produces alerts. Alerts are the input to Security Copilot. A well tuned detection layer sends Copilot high fidelity signals with rich context. A poorly tuned layer floods Copilot with noise and slows investigation.

Detection quality determines how useful Security Copilot can be. Layer 04: Security Copilot takes the alerts produced here and builds investigation timelines from them.
