Layer 02

Telemetry Collection

Turning endpoint activity into structured, actionable signals.

Why telemetry is the foundation of detection

Detection is only as good as the data beneath it. In an enterprise environment, endpoints generate thousands of events per hour — most of them noise. The telemetry layer is responsible for identifying which signals matter, ensuring they are collected consistently, and routing them to the right analytics platform. Without high-fidelity telemetry, detection rules produce false negatives, investigations lack evidence, and Security Copilot has nothing to reason about.

Primary telemetry sources

Windows Event Logs

The native audit trail of Windows activity. Security channel logs authentication events (4624, 4625, 4648), process creation (4688), and privilege use. System and Application channels surface service failures and software behavior.

PowerShell Logging

Script block logging (Event 4104) and module logging capture the full content of executed PowerShell — critical for detecting obfuscated commands, living-off-the-land attacks, and malicious automation.

Microsoft Defender Telemetry

Defender for Endpoint streams process trees, network connections, file modifications, and behavioral detections to Microsoft's cloud backend, enriching alert context beyond what local logs provide.

Azure Diagnostics & Sign-in Logs

Entra ID logs every authentication attempt, Conditional Access evaluation, and token issuance. Azure Activity Logs record control-plane changes to resources — VM restarts, role assignments, policy modifications.

Hardware & Firmware Events

HP iLO, IPMI, and UEFI logs surface physical layer events — unexpected reboots, hardware failures, and firmware changes that OS-level monitoring cannot see.

Network Flow Data

DNS query logs, proxy logs, and NetFlow data reveal lateral movement and command-and-control communication patterns that endpoint-only telemetry misses.

Telemetry quality: the signal-to-noise problem

The challenge: a 50,000-endpoint environment generates millions of events per day. Collecting everything is not the goal — collecting the right things at the right fidelity is. Discuss audit policy tuning (enabling process creation logging, command line auditing), log forwarding architecture (Windows Event Forwarding vs. agent-based collection), and the difference between raw log volume and detection-relevant signal density.

How telemetry feeds the pipeline

Telemetry is the fuel for every layer above it. Detection rules in Sentinel and Defender operate on this data. Security Copilot uses telemetry as evidence when building investigation timelines. Automated response playbooks query telemetry to validate findings before executing remediation.
