Use Case 01

Copilot-Assisted Investigation

1

Problem

Most security teams know this pattern. An alert comes in for something like a weird PowerShell command, and now an analyst has to go figure out what actually happened. They open Sentinel, dig through sign in logs, check if Defender saw anything related on that machine, and write up notes as they go. None of this is hard, but it takes time, and a busy team is fielding alerts like this constantly. The work is mostly repetitive, which makes it a good candidate for some help from automation.

2

Signal

Here is what a real version of this looks like. Defender for Endpoint flags an obfuscated PowerShell command running on someone's laptop. Around the same time, Sentinel notices that same user had a risky sign in about half an hour earlier, maybe from a new location or a new device. Defender XDR sees that these two things are probably related and bundles them into one incident instead of two separate alerts.

3

Action

Instead of letting the analyst start from a blank page, the team gives Copilot a specific, repeatable prompt to run against incidents like this. The prompt tells Copilot which incident to look at, what time window matters, and asks for three things: a plain summary of what happened, which MITRE ATT&CK technique it matches, and which user and machine were involved. Because the prompt is the same every time, the output is consistent, and the team keeps a record of which prompt produced which result.

4

Outcome

The analyst still makes the call on what to do next, but they are not starting from zero. They get a clear summary instead of raw logs, which cuts down the time spent just understanding what happened. Because every summary is tied back to a known prompt, it is also easy to check later whether the prompt is still doing a good job or needs adjusting. That matters more than it sounds. Teams that skip this step end up with Copilot outputs nobody can explain or trust later.
