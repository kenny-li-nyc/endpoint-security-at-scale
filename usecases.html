Use Case 01

Copilot-Assisted Investigation

Analysts lose time piecing together context after an alert fires. A single PowerShell alert requires opening Sentinel, querying sign-in logs, checking Defender for related activity, and writing notes by hand. At enterprise scale this repeats hundreds of times a day.

Signal

A Defender for Endpoint alert flags an obfuscated PowerShell command on a managed workstation. Sentinel correlates it with a risky sign-in event from the same user thirty minutes earlier. Defender XDR groups both into a single incident.

Action

A standardized prompt template asks Copilot to summarize the incident, map the technique to MITRE ATT&CK, and list the affected identity and host. The prompt specifies the incident ID and time window so Copilot queries only relevant telemetry. The output gets logged alongside the incident for auditability.

Outcome

The analyst reviews a structured summary instead of building one from scratch. Investigation time drops because correlation work is already done. Every Copilot output is tied to a specific prompt template, which makes the process auditable and repeatable across the team.
