Layer 01

Windows & Azure Platform

The identity and OS foundation that every security control depends on.

The role of the platform layer

In an enterprise with 50,000+ endpoints, the Windows and Azure platform is not just infrastructure — it is the enforcement surface for every security policy. Group Policy, Entra ID Conditional Access, and Active Directory trust boundaries determine what users can do, what devices can connect, and which security controls apply. A misconfigured platform layer undermines every layer above it.

Core components

Active Directory

The authoritative identity store for on-premises environments. Controls authentication, authorization, and group membership across domain-joined endpoints.

Microsoft Entra ID

Cloud identity provider extending AD to Azure and SaaS applications. Enables Conditional Access policies, MFA enforcement, and device compliance checks.

Group Policy (GPO)

The primary mechanism for enforcing security baselines across Windows endpoints — controlling audit settings, PowerShell execution policy, firewall rules, and software restrictions.

Windows Internals

Understanding how Windows handles processes, services, authentication tokens, and the registry is essential for distinguishing normal system behavior from attacker tradecraft.

Azure Arc

Extends Azure management and policy enforcement to on-premises and multi-cloud Windows servers, enabling unified governance across hybrid environments.

Entra ID Conditional Access

Policy engine that evaluates device compliance, user risk, and location signals before granting access — acting as a real-time access control checkpoint.

Why platform hygiene matters for security

Attackers target the platform layer because it offers the highest leverage: a compromised AD admin account or a misconfigured GPO can disable security tools across thousands of endpoints simultaneously. Reference common attack patterns: pass-the-hash, Kerberoasting, GPO abuse. Explain that hardening the platform layer — enforcing least privilege, auditing privileged group membership, enabling Protected Users security group — reduces the blast radius of any breach above it.

Platform layer in the enterprise pipeline

This layer feeds Layer 2 (Telemetry) because Windows authentication events, GPO application logs, and Entra ID sign-in logs are primary telemetry sources. A well-configured audit policy ensures those signals are generated consistently across all endpoints.
